= 9. Architecture Decisions

Status: implementation-informed.

ADR bodies are owned by adr-writer and are not generated in this chapter. This chapter indexes existing ADRs and highlights missing decision records.

== 9.1 ADR Index

[cols="1,1,3",options="header"]
|===
| ADR | Status | Notes

| link:../adr/0001-use-content-addressed-yaml-for-certificate-profiles.md[ADR 0001 — Use content-addressed YAML for certificate profiles]
| Accepted
| Certificate profiles are immutable, content-addressed YAML resources. The accepted decision includes the normative CertificateProfile v1 schema/admission, normalization, RFC 8785 JCS/SHA-256 identity, whole-string ID grammar, golden fixture, versioning, and identity-versus-issuance-policy boundary. Implementation remains incomplete.

| link:../adr/0002-use-git-backed-governed-event-store-for-certificate-profile-lifecycle.md[ADR 0002 — Use a Git-backed governed event store for certificate profile registry lifecycle]
| Accepted architectural direction / not implemented
| Git is selected only as a governed, low-throughput source for certificate profile registry/lifecycle events and immutable artifact/manifest publication. CQRS keeps read models replaceable, and the Event Sourcing path remains open through structured event envelopes and replayable projections. No Git event-store implementation or projectors/read models exist yet.
|===
